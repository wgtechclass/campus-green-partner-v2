6. 不使用學生個人帳號、不使用 GPS、不強制串接 AI API 的 MVP 原則

## 1. 專案概述

### 1.1 專案名稱

正式課程與系統名稱：

**《校園綠夥伴：環境復甦任務》**

學生端遊戲化呈現名稱：

**《淨芽小隊：校園健康冒險》**

### 1.2 專案目的

本系統是一款結合真實校園行動、健康走動、垃圾辨識分類與虛擬角色養成的網頁式任務遊戲，適用於國小高年級四週課程。

學生以小組為單位，組成「淨芽小隊」，透過接受任務、走出教室、累積步數、發現垃圾或環境問題、拍照紀錄、辨識分類、撿拾整理與成果倡議，累積不同精靈的能量，使小隊升級、解鎖地圖，最後完成「淨芽小隊守護報告」。

### 1.3 課程定位

本系統支援：

- 校訂彈性學習課程
- 綜合活動課程
- 環境教育主題課程
- 健康促進課程
- 資訊教育與 AI 素養課程
- 國小高年級跨域探究課程

### 1.4 適用對象

| 對象 | 說明 |
|---|---|
| 學生 | 國小五、六年級學生 |
| 教師 | 導師、綜合活動教師、彈性課程教師或行政推動者 |
| 使用單位 | 單班、高年級、年級群或全校高年級 |

## 2. 核心玩法

### 2.1 新版遊戲核心循環

系統需強化為以下遊戲循環：

```text
接受任務
↓
走出教室
↓
累積步數／探索能量
↓
遇見垃圾或校園環境問題
↓
拍照紀錄與辨識
↓
撿拾與分類
↓
淨芽精靈獲得能量
↓
小隊升級
↓
解鎖新地圖／新任務
↓
再次探索
```

### 2.2 玩家單位

本遊戲採用小組制。

| 項目 | 設定 |
|---|---|
| 每組人數 | 4～5 人 |
| 每組單位 | 一支淨芽小隊 |
| 每組角色 | 共同養成一組淨芽精靈，並可選一位隊長精靈 |
| 登入方式 | 選擇班級、組別、輸入隊名 |
| 不採用 | 個人帳號、密碼、GPS 定位 |

### 2.3 遊戲目標

學生在四週內完成：

1. 建立淨芽小隊
2. 走出教室並完成校園探索
3. 發現校園垃圾或環境問題
4. 完成垃圾辨識與分類任務
5. 累積能量並讓小隊升級
6. 解鎖校園地圖與任務徽章
7. 完成校園環境改善提案與宣導成果

## 3. 遊戲世界觀

校園中住著一群「淨芽精靈」，牠們守護校園的健康、乾淨與活力。

最近校園角落出現了髒亂能量，讓淨芽精靈的力量變弱。學生需要組成淨芽小隊，接受 AI 小隊長發布的任務，走出教室探索校園，累積健康步數，發現垃圾並完成辨識分類，讓淨芽精靈恢復能量。

當小隊完成任務，精靈會升級，地圖會逐步解鎖。最後，學生要用自己的觀察與行動完成「淨芽小隊守護報告」，讓校園變得更健康、更乾淨、更美麗。

## 5. 淨芽小隊角色系統

### 5.1 角色定位

系統原本的「綠夥伴」角色需擴充為「淨芽小隊角色」。

建議保留既有 `Pets` 資料表與 `selectedPetId` 欄位，但將角色內容替換或擴充為六位淨芽精靈。

### 5.1.1 角色圖片與素材規範

六位淨芽精靈必須直接使用使用者提供之角色圖作為正式遊戲素材。實作時需從參考圖中分別裁切綠芽、水滴、陽光、紫寶、心心、阿栗，製作成獨立透明背景 PNG，供首頁、小隊建立頁、小隊基地、任務卡與守護報告使用。

角色素材不得只用文字、emoji、placeholder、簡化色塊或手繪 SVG 假角色替代。參考圖可以拆成角色與 UI 小素材使用，但不可將整張參考圖直接嵌入遊戲畫面。

每張角色圖需符合：

1. 透明背景。
2. 正方形畫布。
3. 角色完整置中，頭、身體、手腳與代表配件不可被裁掉。
4. 不可帶入參考圖背景、說明文字、邊框或隔壁角色碎片。
5. 在不同容器中以 `object-fit: contain` 類似策略完整顯示，不得因放大造成爆版或裁切。

### 5.2 六位淨芽精靈

| 精靈 | 角色稱號 | 功能定位 | 對應能力值 |
|---|---|---|---|
| 綠芽 | 環境守衛 | 發現垃圾、綠化校園、守護環境 | 偵查值、淨化值 |
| 水滴 | 活力守衛 | 提醒喝水、補充能量、維持健康 | 體力值 |
| 陽光 | 運動守衛 | 鼓勵走動、累積步數、推動探索 | 體力值、影響力 |
| 紫寶 | 智慧守衛 | AI 辨識、任務分析、路線建議 | 智慧值 |
| 心心 | 情緒守衛 | 鼓勵同伴、團隊合作、互相提醒 | 影響力 |
| 阿栗 | 回收守衛 | 垃圾分類、資源回收、分類挑戰 | 淨化值、智慧值 |

### 5.3 隊長精靈

每組建立小隊時，需選擇一位「隊長精靈」。

隊長精靈會出現在：

1. 首頁
2. 小隊基地
3. 任務卡
4. 成長頁
5. 守護報告

### 5.4 其他精靈解鎖

系統可採兩種版本：

| 版本 | 設計 |
|---|---|
| V2.0 簡化版 | 六位精靈一開始全部顯示，隊長精靈為主要代表 |
| V2.1 進階版 | 隨任務完成逐步解鎖其他精靈 |

建議先做 V2.0，避免牽動資料邏輯過大。

## 6. 能力值與能量系統

### 6.1 保留現有能力值欄位

為降低修改成本，資料欄位可先沿用現有能力值：

| 既有欄位 | 顯示名稱 | 新版詮釋 |
|---|---|---|
| `stamina` | 體力值 | 走出教室、累積步數、完成校園探索 |
| `scout` | 偵查值 | 發現垃圾熱點或校園環境問題 |
| `cleanse` | 淨化值 | 撿拾、整理、分類、提出改善行動 |
| `wisdom` | 智慧值 | 垃圾辨識、分類判斷、原因分析 |
| `influence` | 影響力 | 團隊合作、宣導作品、行動承諾 |

### 6.2 新增顯示型能量名稱

前端畫面可使用更有遊戲感的名稱，但內部欄位仍對應既有資料。

| 遊戲顯示名稱 | 內部欄位 |
|---|---|
| 健康能量 | `stamina` |
| 探索能量 | `scout` |
| 淨化能量 | `cleanse` |
| 智慧能量 | `wisdom` |
| 守護能量 | `influence` |

### 6.3 步數資料

需新增步數相關欄位或在任務回報中加入步數欄位。

| 欄位 | 類型 | 說明 | 必填 |
|---|---|---|---|
| `stepCount` | 數字 | 本次任務累積步數，可由學生手動輸入或教師指定 | 否，建議第 2 週必填 |
| `stepTarget` | 數字 | 本次任務目標步數，預設可為 800、1000 或教師自訂 | 否 |
| `routeName` | 文字 | 任務路線名稱，例如操場東側、川堂路線 | 否 |

> V2.0 不強制串接 iPad 健康資料或 GPS。步數可以先採手動輸入、任務完成勾選或教師指定數值。

## 8. 任務卡系統

### 8.1 任務卡目的

任務卡需讓學生清楚知道「今天要去哪裡、做什麼、完成後會獲得什麼」。

### 8.2 任務卡欄位

| 欄位 | 說明 |
|---|---|
| 任務名稱 | 例如「操場東側任務」 |
| 任務地點 | 校園區域或路線 |
| 任務目標 | 例如走 800 步、發現 1 個垃圾熱點、完成分類 |
| 任務步驟 | 1～3 個明確步驟 |
| 獎勵能量 | 完成後獲得哪些能力值 |
| AI 小隊長提示 | 一句任務提醒或健康提醒 |
| 完成狀態 | 未開始、進行中、已完成 |

### 8.3 任務卡範例

```text
任務名稱：操場東側偵查
任務地點：操場東側與跑道周邊
任務目標：完成 800 步探索，發現並紀錄 1 個環境問題
任務步驟：
1. 小組一起走到操場東側
2. 觀察是否有垃圾或髒亂能量點
3. 完成照片說明與垃圾分類回報
獎勵：體力值 +10、偵查值 +10、智慧值 +5
AI 小隊長提示：走路時請注意安全，也記得觀察垃圾出現的位置。
```

## 9. 垃圾辨識與分類任務

### 9.1 任務定位

垃圾辨識分類是新版核心玩法之一。學生不只回報垃圾類型，還要完成「辨識 → 分類 → 說明」的學習流程。

### 9.2 垃圾辨識頁需求

任務回報頁需增加「垃圾辨識區塊」。

| 欄位 | 類型 | 說明 |
|---|---|---|
| `photoUrl` | 文字或上傳連結 | 任務照片連結，沿用現有欄位 |
| `photoNote` | 文字 | 照片說明，沿用現有欄位 |
| `trashTypes` | 多選 | 垃圾類型，沿用現有欄位 |
| `classificationType` | 單選 | 一般垃圾、資源回收、廚餘、有害垃圾、其他 |
| `classificationReason` | 文字 | 為什麼這樣分類 |
| `aiGuessNote` | 文字 | AI 或小組辨識結果說明，可選 |

### 9.3 垃圾分類選項

原垃圾類型保留，並新增分類結果。

#### 垃圾類型

- 紙類
- 塑膠袋
- 飲料杯
- 吸管或杯套
- 食物包裝
- 鉛筆屑或橡皮擦屑
- 寶特瓶
- 其他

#### 分類結果

- 一般垃圾
- 資源回收
- 廚餘
- 有害垃圾
- 不確定，需要查詢

### 9.4 分類小挑戰

V2.0 可先以簡易選擇題呈現，不需拖曳。

每次任務提交前，可出現一題分類問題：

> 「寶特瓶應該放在哪一類？」

學生選擇後，系統顯示簡短回饋。

V2.1 可再改為拖曳式分類小遊戲。

## 10. AI 小隊長功能

### 10.1 AI 小隊長定位

AI 小隊長是遊戲中的引導 NPC，不一定代表已串接 AI API。

第一版可以先用固定文案與教師端 AI prompt 實作，第二版再考慮 API。

### 10.2 AI 小隊長功能

| 功能 | V2.0 實作方式 | V2.1 延伸 |
|---|---|---|
| 發布每日任務 | 固定任務卡文案 | 依資料自動生成任務 |
| 推薦探索路線 | 依教師設定區域顯示 | 根據熱點資料推薦路線 |
| 統計步數與活動量 | 顯示學生輸入步數 | 串接裝置或健康資料 |
| 垃圾辨識與分類教學 | 顯示分類提示與選項 | 圖像辨識或 AI 分析 |
| 健康小提醒 | 固定提醒語 | 依步數或任務狀態生成 |
| 生成任務回饋 | 固定規則回饋 | AI 依小組資料生成回饋 |
| 產生守護報告摘要 | 教師端複製 AI prompt | API 自動摘要 |

### 10.3 AI 小隊長提示語範例

- 「今天先從操場開始探索，注意安全，也記得觀察垃圾容易出現的位置。」
- 「你們已經累積不少探索能量，下一步試著判斷垃圾可以怎麼分類。」

## 18. AI 分析 Prompt 修正版

教師端「複製給 AI 分析」功能需改為包含步數、分類、精靈能量與守護報告。

```text
你是一位國小高年級環境教育、健康促進與遊戲化學習課程顧問。以下是學生在《校園綠夥伴：環境復甦任務／淨芽小隊：校園健康冒險》中提交的資料。請協助整理：

1. 哪些校園區域最常出現垃圾或環境問題？
2. 學生累積步數與校園探索情形如何？
3. 最常見的垃圾類型有哪些？
4. 學生的分類判斷是否合理？有哪些需要提醒？
5. 學生提出的可能原因有哪些共通點？
6. 學生提出的改善方法中，哪些最具體、最可行？
7. 各小隊的精靈能量或能力值可以給予什麼回饋？
8. 可以提供哪些適合國小學生理解的健康與環境提醒？
9. 可以整理成哪些全校宣導重點？
10. 請生成一段可放入「淨芽小隊守護報告」的成果摘要。

請用繁體中文、國小高年級能理解的語氣，並保持鼓勵、具體、可行。
```

## 20. 給 AI Coding 工具的修改重點摘要
