# SPEC v1 → v2 前後差異對照表

> 對照目的：協助在現有《校園綠夥伴：環境復甦任務》MVP 基礎上，依據新提供的兩張遊戲玩法圖進行修正。  
> 原則：不砍掉重練，而是保留現有架構，增加「淨芽小隊」、「步數健康冒險」、「垃圾辨識分類」、「地圖解鎖」、「AI 小隊長」等玩法。

---

## 一、整體方向差異

| 項目 | v1 原設計 | v2 修正後設計 | 修改影響 |
|---|---|---|---|
| 課程／系統名稱 | 校園綠夥伴：環境復甦任務 | 正式名稱保留；學生端遊戲層新增「淨芽小隊：校園健康冒險」 | 低，主要改文案與首頁 |
| 核心定位 | 環境行動＋虛擬寵物養成 | 校園探索＋健康步數＋垃圾辨識分類＋精靈養成 | 中，需新增步數與分類任務 |
| 遊戲感 | 任務、能力值、徽章、成果卡 | 任務卡、校園地圖、步數能量、精靈小隊、地圖解鎖 | 中，需 UI 強化 |
| 最終產出 | 校園綠夥伴成果卡 | 淨芽小隊守護報告 | 低到中，欄位需增加 |
| AI 角色 | 教師端分析 prompt | AI 小隊長：任務發布、路線提醒、分類提示、成果摘要 | 中，V2.0 可先固定文案 |
| 角色圖像素材 | 可簡化或暫用 placeholder | 六位角色需由提供角色圖裁切、去背為正式透明 PNG，不以文字、emoji、placeholder 或假 SVG 替代 | 中，需 assets 與 UI 容器 |
| 學生端視覺風格 | 一般任務工具式 UI | 國小學生取向的遊戲化、活潑可愛 UI，但仍需清楚可讀 | 中，需首頁、角色卡、地圖、基地與守護報告視覺調整 |

## 二、遊戲循環差異

| 項目 | v1 原循環 | v2 新循環 | 修改影響 |
|---|---|---|---|
| 遊戲流程 | 接任務 → 走校園 → 完成觀察 → 回報資料 → 獲得能力值 → 成長 → 成果卡 | 接受任務 → 走出教室 → 累積步數 → 遇見垃圾 → 拍照辨識 → 撿拾分類 → 精靈獲得能量 → 精靈升級 → 解鎖新地圖 → 再次探索 | 中，高度影響首頁、地圖頁、任務頁文案 |
| 走校園 | 文字描述為主 | 明確設計成「步數／探索能量」 | 中，需新增 `stepCount` |
| 遇見垃圾 | 記錄垃圾類型 | 拍照辨識、分類判斷、分類理由 | 中，需新增分類欄位 |
| 解鎖機制 | 四週任務自然推進 | 地圖區域、小隊等級、徽章逐步解鎖 | 中，需 UI 呈現 |

## 三、角色系統差異

| 項目 | v1 原設計 | v2 修正後設計 | 修改影響 |
|---|---|---|---|
| 角色名稱 | 綠夥伴 | 淨芽小隊精靈 | 低到中，需改角色資料 |
| 角色數量 | 五隻：小葉靈、步步獸、淨淨鳥、亮光鹿、花芽龍 | 六位：綠芽、水滴、陽光、紫寶、心心、土豆 | 中，需更新 `Pets` 預設資料 |
| 角色功能 | 各自對應較高能力值 | 每位精靈有明確任務功能：環境、健康、運動、AI、情緒、回收 | 中，需更新 UI 說明 |
| 角色圖片 | 可用簡化圖或 placeholder | 使用提供角色圖裁切去背後的透明 PNG，角色需完整置中、不帶背景或其他角色碎片 | 中，需新增 `assets/spirits/*.png` |
| 小隊養成 | 每組養一隻綠夥伴 | 每組可選一位隊長精靈，並呈現淨芽小隊概念 | 中，若只做隊長精靈則不需大改資料表 |

## 十一、開發優先順序建議

### P0：一定要先改

1. 首頁與文案改成「校園綠夥伴＋淨芽小隊」雙層名稱。
2. 更新角色資料為六位淨芽精靈。
3. 裁切並去背六位角色圖，接入角色卡、首頁、小隊基地、任務地圖與守護報告容器。
4. 任務回報新增步數、分類結果、分類理由。
5. 更新計分邏輯。
6. 成果卡改名為守護報告。
7. 教師端回報與統計欄位同步更新。

### P1：建議第二步改

1. 任務地圖改成地圖解鎖感。
2. 成長頁顯示小隊等級。
3. 新增任務卡視覺。
4. 新增 AI 小隊長提示語。
5. 新增步數與分類徽章。

### P2：之後再做

1. 拖曳式垃圾分類小遊戲。
2. 真實校園平面圖。
3. QR Code 任務點。
4. AI 圖像辨識。
5. 精靈動畫與進化圖。

## 十二、給 Claude Code / Codex 的摘要指令

請在現有 `campus-green-partner` 專案上進行增量修改，不要重建整個專案。

修改目標是將現有《校園綠夥伴：環境復甦任務》MVP 升級為《校園綠夥伴：環境復甦任務》v2，學生端遊戲層加入《淨芽小隊：校園健康冒險》玩法。

請優先處理：

1. 新增「淨芽小隊」遊戲名稱、首頁標語與核心循環說明。
2. 將角色資料更新為六位精靈：綠芽、水滴、陽光、紫寶、心心、土豆。
3. 將提供角色圖裁切、去背為六位正式透明 PNG，放入 `assets/spirits/*.png`，不得使用整張參考圖、placeholder、emoji、文字卡或假 SVG 取代。
4. 任務回報表新增步數、路線、分類結果、分類理由、AI 小隊長提示。
5. 任務地圖加入任務卡與地圖解鎖感。
6. 成長頁加入小隊等級：萌芽級、成長級、茁壯級、守護級。
7. 成果卡改為「淨芽小隊守護報告」。
8. 教師端增加步數統計、分類結果統計、守護報告檢視與新版 AI prompt。
